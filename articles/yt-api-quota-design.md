YouTubeのキーワードリサーチツールを個人開発して公開しています。YouTube Data API v3 の無料クォータは **1日10,000ユニット（プロジェクト単位）**。個人の検証なら十分ですが、不特定多数に公開するWebサービスとなると、何も考えずに作れば初日に枯渇します。

この記事は、無料枠のまま公開サービスを運用するために実際に入れたクォータ設計のメモです。バックエンドは Cloudflare Workers + KV です。

## まず敵を知る: APIコスト表

YouTube Data API はエンドポイントごとにコストが違います（2026年6月時点で確認）。

| エンドポイント | コスト | 備考 |
|---|---:|---|
| search.list | **100 units** | 圧倒的に高い |
| videos.list | 1 unit | 1回で最大50ID |
| channels.list | 1 unit | 1回で最大50ID |

支配項は明らかに `search.list` です。キーワード分析1回の構成は「search 1回 + 動画50件のvideos.list 1回 + チャンネル50件のchannels.list 1回」で **102ユニット**。つまり素朴に作ると **1日約98回の分析でクォータが尽きます**。全世界に公開するサービスとしては心もとない数字です。

## 設計1: search結果はKVで48時間キャッシュし、全ユーザーで共有する

同じキーワードは誰が検索しても同じ結果でいい、というのがリサーチツールの性質です。そこで search の結果を Cloudflare KV に**48時間キャッシュし、ユーザー間で共有**しました。

- 人気キーワードほどキャッシュヒット率が上がる（2人目以降はクォータ消費ゼロ）
- チャンネル統計は鮮度を優先して24時間
- 急上昇ランキングは1日に数回しか更新しないので24時間キャッシュ・全ユーザー共有で**1日約2ユニット**

なお、APIの利用規約上キャッシュには上限期間（30日）があります。48時間/24時間はそこに余裕をもって収まる設定です。

## 設計2: バッチを最大まで詰める

`videos.list` と `channels.list` は1回の呼び出しで50IDまで指定できます。必要な呼び出し回数は純関数にして、テストで固定しました。

```typescript
export const QUOTA_COST = { search: 100, videoList: 1, channelList: 1 } as const;
export const DAILY_QUOTA_LIMIT = 10_000;
export const LIST_BATCH_SIZE = 50;

export function batchCalls(idCount: number, batchSize = LIST_BATCH_SIZE): number {
  if (idCount <= 0) return 0;
  return Math.ceil(idCount / batchSize);
}
```

1件ずつ `videos.list` を呼ぶ実装にしてしまうと、同じ情報量で50倍のユニットを払うことになります。

## 設計3: 日次クォータカウンタをKVに置き、枯渇を「エラー」ではなく「状態」にする

API側で429が返ってから慌てるのではなく、**自分でユニット消費を予約制にしました**。

1. リクエストが来たら、まず「この分析にかかるコスト」を計算する
2. KV上の日次カウンタと突き合わせ、上限を超えるなら API を呼ばずに `QUOTA_EXHAUSTED` を返す
3. 実行したら消費分をカウンタに加算（TTL 2日で自動失効するので日付ロールオーバーの掃除が不要）

フロント側は `QUOTA_EXHAUSTED` を受けたら「本日の分析上限（サーバー側）に達しました」と表示するだけ。枯渇が異常系ではなく仕様上の状態になるので、運用の心配事がひとつ減ります。

## 設計4: ユーザー単位のレート制限で「ひとりに食い尽くされない」ようにする

プロジェクト全体のクォータを守っても、1人のヘビーユーザー（やbot）が全部使ったら他の人が使えません。そこで匿名ID（またはライセンスキー）ごとに**無料3回/日、Pro 30回/日**のレート制限を入れました。

カウンタはこれもKVです。Workers + KVだけで完結するので、レート制限のためにRedisを立てたりはしていません。

## 全体像

```
ユーザー → Cloudflare Worker
            ├─ レート制限チェック（ユーザー単位: 無料3回/Pro30回）
            ├─ KVキャッシュ確認（キーワード48h / チャンネル24h, 全ユーザー共有）
            │    └─ ヒット → クォータ消費ゼロで返却
            ├─ 日次クォータ事前チェック（足りなければ QUOTA_EXHAUSTED）
            └─ YouTube Data API 呼び出し → KVに保存 → 消費を加算
```

この構成で、無料枠1万ユニット/日のまま公開運用できています。クォータに効く順で言うと「**キャッシュ共有 > バッチ詰め > 事前チェック > ユーザー別制限**」でした。

## おわりに

このクォータ設計で動いているのが「[ツベリサーチ](https://tsube-research.pages.dev)」です。キーワードを入れると競合動画・参入チャンスを分析します。登録不要・無料で1日3回まで試せるので、よければ触ってみてください。
